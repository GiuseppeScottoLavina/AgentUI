Dialog / Modal

When to use

Custom content dialogs
Forms, complex layouts, multi-step wizards, or any non-standard content
Details or previews
Image galleries, product details, user profiles, expanded views
Modal forms with validation
Login, settings, feedback, where you need full control over layout

When NOT to use

Don't use for simple yes/no confirmations — use <au-confirm> instead (less code, built-in patterns) Don't use for temporary notifications — use showToast() instead (auto-dismisses, non-blocking) Don't use for persistent status messages — use <au-alert> inline instead (always visible)

Accessibility

Modal uses role="dialog" and traps focus inside. Press Escape or click backdrop to close. Focus returns to trigger on close.
