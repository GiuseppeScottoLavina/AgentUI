Alert

<au-alert> displays contextual feedback messages to users. Supports 4 severity levels with semantic colors.

OVERVIEW API STYLING EXAMPLES
Operation completed successfully Here is some helpful information Warning: Please review before proceeding Error: Something went wrong

When to use

Form validation errors
Errors that user must fix before proceeding — always visible
Important contextual information
Warnings about data, requirements, or status that must stay visible
Page-level status
"Feature is beta", "System maintenance soon", "Premium feature"

When NOT to use

Don't use for success feedback after actions — use showToast() instead (disappears automatically) Don't use for confirmations before actions — use <au-confirm> which blocks until user responds

Attributes

Alert type: 'success' | 'info' | 'warning' | 'error' Whether alert can be dismissed

Events

Emitted when alert is dismissed

CSS Custom Properties

Success background Info background Warning background Error background
